This bug occurs when using Tailwind CSS's `@apply` directive with a class that contains a pseudo-selector like `hover`, `focus`, or `active`. The `@apply` directive will only apply the styles for the base state and ignore the pseudo-selector states. For instance: ```html ``` This will only apply the base styles of the `hover:bg-blue-500` class, and the hover effect won't work. The solution is to apply the hover styles separately.
